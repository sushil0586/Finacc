# GST Reports User Guide

Date: 2026-05-08

## Purpose

This guide explains GST Reports in simple business language.

It is meant for finance users, tax users, reviewers, and support teams who need to understand:
- what each GST report screen is for
- when to use each screen
- what to verify before filing or exporting

## What GST Reports Mean

GST Reports are the reporting layer that helps your team review GST data already recorded in purchase and sales transactions.

These screens do not create GST values by themselves. They summarize and organize the values that come from posted business transactions.

In simple terms:
- Sales invoices feed outward tax reporting
- Purchase invoices feed inward tax reporting
- GST report screens help review, validate, reconcile, and export that information

## Main GST Report Screens

### 1. GSTR-1

Use this screen for outward supply reporting.

It is mainly used to:
- review B2B, B2C, export, and note data
- inspect section-wise sales values
- review invoice-level drilldown
- identify warnings before export or filing preparation

Typical users:
- sales tax reviewers
- indirect tax accountants
- finance managers validating outward tax numbers

### 2. GSTR-3B

Use this screen for monthly summary-style GST review.

It is mainly used to:
- review output liability
- review ITC position
- review key payable and offset-ready values
- export the report for working papers or reconciliation

Typical users:
- tax operations teams
- compliance managers
- finance reviewers preparing returns

### 3. GSTR-9

Use this screen for annual GST reporting and annual review.

It is mainly used to:
- summarize yearly outward and inward values
- review annual tax totals across report tables
- validate annual consistency
- freeze annual snapshots before filing workflow steps

Typical users:
- year-end finance reviewers
- indirect tax specialists
- audit and compliance teams

## Basic Working Flow

The usual GST reporting flow is:

1. Post purchase and sales transactions properly.
2. Select the correct entity, financial year, branch, and date range.
3. Open the required GST report.
4. Review summary totals.
5. Review section or table details.
6. Check validations and warnings.
7. Export the report if needed.
8. Use the output for return working, reconciliation, or filing support.

## Before Using GST Reports

Make sure these basics are correct:
- the user has access to the right entity
- the correct financial year is selected
- the correct branch or subentity is selected if applicable
- invoices are posted, not left in draft
- GSTIN, place of supply, tax regime, and tax amounts are properly captured in source documents

If any of the above is wrong, the report may still open but the values may not be reliable.

## How to Use GSTR-1

### What to Review First

Start with:
- overall summary
- section-wise totals
- warning count

Then move into:
- B2B
- B2CL or B2CS
- exports
- debit notes and credit notes
- invoice drilldown where needed

### Common Things to Check

- customer GSTIN correctness
- place of supply correctness
- inter-state vs intra-state treatment
- note linkage to original invoices
- export treatment

### When to Export

Export when:
- you need working papers
- you need review evidence
- you want offline analysis
- you want management signoff support

## How to Use GSTR-3B

### What to Review First

Start with:
- summary cards or totals
- payable values
- ITC values
- tax breakup

### Common Things to Check

- output liability looks reasonable for the month
- ITC values align with purchase and statutory expectations
- major month-on-month spikes are explained
- export values match finance review expectations

## How to Use GSTR-9

### What to Review First

Start with:
- annual summary
- implemented tables
- validations

Then review:
- yearly totals by table
- notes and adjustments
- annual tax consistency

### Freeze and Filing Support

If your organization uses the freeze process:
- freeze only after review is complete
- treat each frozen version as a reporting checkpoint
- use the latest approved freeze for final working

## Common User Mistakes

### Wrong entity or branch selected

This is one of the most common mistakes.

Always confirm:
- entity
- financial year
- branch or subentity
- period

before reviewing totals.

### Expecting reports to fix bad transactions

GST Reports reflect source data. If invoices are wrong, the report will also be wrong.

### Reviewing draft data

Unposted or incomplete source transactions can cause confusion. Review only after operational posting is complete.

### Ignoring warnings

Warnings do not always mean the report is unusable, but they should always be reviewed before signoff.

## Troubleshooting Basics

If the report looks wrong:

1. Check entity and period filters.
2. Check whether transactions are posted.
3. Check customer or vendor GST details.
4. Check tax regime and place of supply.
5. Check whether the user has opened the correct GST report.

If the screen does not open:

1. check role permission
2. check entity access
3. log out and back in if access was recently changed

## Simple FAQ

### Does GST Reports create GST entries?

No. It only reports on GST data already created by source transactions.

### Which report should I use monthly?

Usually GSTR-1 and GSTR-3B.

### Which report should I use yearly?

GSTR-9.

### If GSTR-1 looks wrong, where should I check first?

Check posted sales invoices, GSTIN, tax regime, place of supply, and invoice classification.

### If GSTR-3B looks wrong, where should I check first?

Check posted sales and purchase data, ITC treatment, and the selected period.

### If GSTR-9 looks wrong, where should I check first?

Check the yearly scope, annual transaction completeness, and whether the right period and entity were selected.
